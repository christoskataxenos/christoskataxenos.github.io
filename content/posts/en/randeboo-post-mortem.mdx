## 🔄 The Great Pivot: From Silos to Full Stack

We started exactly as classic Software Engineering textbooks recommend: "You do Frontend, you do Backend, you do Testing." 

**The result?** Deadlocks everywhere. The Frontend guy was waiting for the API, the Backend guy was waiting for the database schema, and the Tester was playing solitaire. Every single change required a Viber call with vacuum cleaner noises in the background just to get on the same page.

Realizing that at this rate we would graduate in 2035, I proposed a drastic **pivot**: **Full Stack Module Ownership**.

<Callout type="info" title="The New Strategy">
Each team member took over a complete functional feature (module) and built it from top to bottom: from adapting and extending the database queries for their specific features to the Tkinter UI layout.
</Callout>

This reduced dependencies to zero. Collaboration with Demetris, Panos, and Calliope became asynchronous and highly efficient. Everyone had their own sandbox to break things without bothering the others, and Git stopped looking like a war zone.

## 🚦 SPA Routing: Single Page Architecture in Tkinter

To avoid the chaos of multiple windows popping open (reminiscent of 00s adware), I implemented a custom **Router** in `gui_main.py`. The app behaves like a Single Page Application (SPA).

```python
def open_view(self, name: str) -> None:
    """Central router for swapping panels (SPA)."""
    if name not in self.views:
        logging.warning(f"Unknown view: {name}")
        return

    self._clear_content()
    self.views[name]()   # Dynamically render the panel on the shared canvas

def _clear_content(self):
    """Destroys all widgets in the central Content Frame."""
    for widget in self.content_frame.winfo_children():
        widget.destroy()
```

When the user clicks the sidebar, the router clears the active view and renders the new one in the central frame. Clean, fast, and leak-free.

## 💾 The Unsung Hero: backup.py

It was never requested in the guidelines. But as developers, we know that **data loss is the ultimate sin**. So, I dedicated 6 hours to write an autonomous backup system (`backup.py`).

Every time the application boots, a timestamped copy of the SQLite database is created in `/backups`. If the drive crashes or the user messes up, the system keeps the 5 most recent backups and deletes older ones to save disk space.

## ⚡ The Final Crunch and "Creative" Copy-Paste

Like every software release, the final days were a sprint for survival.
- The Employee module (`gui_employees.py`) had to be written yesterday. Instead of over-engineering, I practiced the sacred art of **DRY (or rather, Copy-Paste-Refactor)**: I took the tried-and-tested logic from the Customer module, renamed the fields, adjusted the queries, and within a few hours, we had a fully functional Employee panel.
- Email dispatch (`email_service.py`): Implemented asynchronously (in a separate thread) to prevent the GUI from freezing during the SMTP handshake. I set up a demo Gmail account, enabled App Passwords, and during the live demo, booking notification emails reached the inbox in less than 3 seconds.
