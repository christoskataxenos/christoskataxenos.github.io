At the Hellenic Open University (EAP), as in any modern computer science school, we are now living in the era of **AI Slop**. It is that wave of "low-effort" code flooding our repositories. Code that looks perfect, runs on the first try, but lacks organic, "human" logic.

That's how the idea for **Papatzis Spotter** was born.

The title "Papatzis" (Greek for a shell game juggler/scammer) isn't accidental. It symbolizes the digital "magic" of LLMs: they produce code that dazzles you on the surface, but if you look closely under the table, you'll see the "juggler" just moving bytes around with statistical precision, without real understanding.

## The Opportunity: Breaking the AI-Paradox

Today, most people try to detect AI using... another AI. This is the **AI-Paradox**: a cat-and-mouse game where one neural network tries to outsmart another.

**Papatzis Spotter** follows a different, Pure Science approach:

1.  **Static Analysis vs Neural Networks:** We use deterministic static analysis instead of black boxes (LLMs).
2.  **Privacy First (Hybrid Offline):** The code never leaves your machine. Analysis happens locally.
3.  **Quality Enforcement:** It's not just a policeman; it's a tool that "immunizes" AI agents with quality rules through **PapatzoSkill**.

## The Detection Logic: The "Geometry" of Code

Detection is based on three main pillars, as defined in our logic model:

### A. Structural Analysis (AST-Based)
Using the **Tree-Sitter** engine, we analyze the Abstract Syntax Tree of the code. AI tends to produce "statistically perfect" distributions.

*   **Node Frequency Distribution:** We look for instruction distributions that too closely match the "textbook" examples the models were trained on.
*   **Depth Variance:** Humans produce irregular depth in their nested structures. AI tends toward a uniform, "flat" complexity that makes sense statistically but not necessarily architecturally.

### B. Statistical Signals & Entropy
*   **Shannon Entropy:** We calculate informational entropy per block. Low entropy means high predictability—a strong signal that the code was generated by a probability model.
*   **Burstiness (Volatility):** Human writing is "explosive." We have large variations in command length and production pace. AI has a steady, flat rhythm.

### C. The "Chaos Factor"
We search for **human chaos**:
*   **Style Drift:** Humans get tired, change their minds, or improve their style within the same project. AI maintains an unnatural, absolute consistency across 100+ files.
*   **Cursed Code:** We recognize local idioms, humor in naming, or "dirty" but clever solutions that an LLM (trained on sanitized documentation) would "politely" avoid.

## The Architecture (The Layered Model)

To achieve speed and security, we built a three-layer model:

1.  **React Frontend:** TypeScript + Vite + Zustand for a blazing-fast UI including the Monaco Editor.
2.  **Tauri Shell:** The Rust bridge that manages secure system communication (IPC).
3.  **Python Sidecar:** The heart of the engine. A standalone executable containing the **Tree-Sitter, Numpy, and Scipy** libraries for heavy statistical processing.

## The Scoring System (Weighted Score)

The final **Slop Score** is not a random guess, but the result of a weighted analysis:

*   **25% AST Uniformity:** Structural "perfection."
*   **20% Entropy Signals:** Token predictability.
*   **20% Project Drift:** Lack of differentiation between modules.
*   **20% Naming & Patterns:** Generic nomenclature.
*   **15% Comment Intensity:** Wikipedia-style explanations that no human would write for themselves.
